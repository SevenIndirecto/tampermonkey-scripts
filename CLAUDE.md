# Tampermonkey scripts repo

Each script lives in its own folder: `<site>-<purpose>/name.user.js` + `README.md`
(+ optional helper scripts). Plain JS, no build step, prefer `@grant none`.

## Userscript conventions

- `@updateURL`/`@downloadURL` point at the raw GitHub master path of the file itself —
  Tampermonkey auto-updates from it, so **bump `@version` on every behavioral change**
  and remember that pushing to master ships to users.
- Commit messages are prefixed with the script name, e.g. `[CM Shipping Estimator] ...`.
- Hardcoded data blocks that are generated by helper scripts sit between
  `// BEGIN GENERATED ...` / `// END GENERATED ...` markers; regenerate, don't hand-edit.

## Cardmarket specifics

- Cloudflare fronts both `www.cardmarket.com` and `help.cardmarket.com`: curl/Node get a
  403 challenge. Any scraping or API calls must run inside a real browser session
  (the user has a logged-in Cardmarket session in Chrome). Throttle to ~1 req/1.5 s.
- Shipping costs API + country-id map (ids are not guessable — verify, don't assume):
  see `cardmarket.com-shipping-estimator/update-shipping-prices.js`.
- Singles page DOM: offer rows are `.article-row` inside `.table-body` (rows are added
  asynchronously — use a MutationObserver, not fixed timers). Seller country exists only
  as localized tooltip text (`data-bs-original-title` / `aria-label`, "Item location: X"),
  set by Bootstrap shortly *after* page load; parsing it is English-only, so `@match`
  should target `/en/` paths.
- `www.cardmarket.com` localStorage can be completely full (a VWO analytics blob has
  been seen occupying the entire quota) — scripts that cache must handle
  `QuotaExceededError`, e.g. fall back to sessionStorage.
- The user-offers Wants filter is a plain GET param: `/en/Magic/Users/<seller>/Offers/
  Singles?idWantslist=<id>` (+`&site=<n>` pagination); total is in `.pagination` text
  ("N Hits"); a zero-hit result drops the pagination bar and wantslist select but keeps
  an empty `#UserOffersTable`.
- Cardmarket rules that matter for estimates: registered shipping is mandatory ≥ 25 €;
  shipping method is the cheapest one whose max value/weight covers the cart; prices on
  the help page include the packaging fee; non-eurozone prices are FX snapshots (~±3%).

## Verifying changes

- Syntax: `node --check <file>`; pure logic can be sanity-checked by eval'ing the data
  block in `node -e`.
- Live check: on the target page, remove the installed script's artifacts (e.g. strip
  `._modified` classes and injected nodes), paste the new script body (without the
  userscript header) into the console, and compare rendered output against known-good
  values before pushing.
